Project 1: 아두이노 디지털 금고 시스템

팀구성: 김민수(PM, 로직 설계), 최승혁(전장 구성)

Problem

기존 키패드 방식은 외부 오염에 취약하고 입력 지연 발생. 특히 다중 암호 처리 시 메모리 병목 현상으로 인한 반응 속도 저하 문제 발견.

Strategy

가변저항(Analog Input)과 스위치 인터럽트 방식을 조합한 다중 입력 알고리즘 설계. 폴링(Polling) 방식 대신 상태 변화 기반 제어로 전환.

Solution

가변저항 0~20 단계별 수치 승인 로직 구현. 3단 암호 교차 검증 시스템 및 RGB-LED/부저를 통한 시각적/청각적 피드백 시스템 구축.

Result

암호 처리 및 검증 속도 0.2초 이내 단축 (60% 개선). 배선 간소화를 통한 기판 노이즈 제로 달성 및 실무형 트러블슈팅 역량 증명.

Why: 현장의 물리적 버튼 내구 한계를 극복하기 위해 비접촉식/회전식 입력 장치의 가능성을 검토.
How: 인터럽트 로직을 적용하여 MCU 부하를 줄이고 실시간성을 확보함.
#MechDesign #TeamLeader

Project 2: 창의적 공학설계 '달리는 로봇' (총괄 팀장)

Problem

초기 조립 후 고속 주행 시 부품 간 간섭 및 동심도 불일치로 인한 직진성 부족 및 주행 중지 현상 빈번.

Strategy

이중 너트 체결 방식 및 축 정렬 정밀 조정을 위한 지그(Jig) 제작 제안. 팀원 간 R&R을 기구 보강과 모션 스터디로 이원화하여 효율 극대화.

Solution

TRIZ 모법 기법을 적용하여 물리적 모순(강성 유지 vs 무게 경량화) 해결. 프레임 보강 및 기어비 최적화로 구동계 안정성 확보.

Result

주행 오차율 15% 개선 및 직진 주행성 98% 확보. 팀 리더로서 협업 갈등을 정밀한 설계 데이터로 설득하며 목표 성능 달성.

Why: 단순 동작을 넘어 미세한 기구적 오차(동심도)가 실제 설비 퍼포먼스에 미치는 영향을 파악하고자 함.
How: 모션 스터디 시뮬레이션을 통해 간섭 지점을 사전 예측하고 하드웨어 보정 수행.
#Python #SensorFusion

Project 3: 파이썬 기반 EV3 지능형 로봇 제어

Problem

복잡한 경로 주행 시 단일 센서의 할루시네이션(오동작) 및 반사율 변화로 인한 경로 이탈 문제 발생.

Strategy

초음파, 자이로, 컬러 센서의 데이터를 가중치 기반으로 융합하는 Kalman Filter 유사 로직 및 파이썬 메모리 관리 최적화 적용.

Solution

센서 데이터 모니터링 모듈 개발. 실시간 자이로 보정을 통한 직선 주행성 강화 및 컬러 센서 샘플링 속도 상향 조절.

Result

경로 정확도 95% 이상 유지. 환경 변화가 극심한 구간에서도 로봇의 자기 안정화 로직이 안정적으로 작동됨을 입증.

Why: 하드웨어의 한계(센서 오차)를 소프트웨어 로직(융합)으로 극복하는 엔지니어링 마인드 함양.
How: 파이썬의 객체지향 프로그래밍을 활용해 센서별 독립 모듈화 구현.
#AI_Architect #VibeCoding

Project 4: AI 기반 풀스택 설비 대시보드 배포

Problem

기존 수동 데이터 기록 방식의 한계로 실시간 설비 모니터링 및 트러블슈팅 이력 공유가 불가능함.

Strategy

Gemini 1.5 Pro를 이용해 '바이브 코딩' 프로세스 도입. AI가 작성한 코드의 보안 및 에러를 엔지니어가 직접 검증(Human-in-the-loop)하는 아키텍처 수립.

Solution

Firebase Realtime DB와 연동된 React 기반 대시보드 구축. Vercel을 통한 자동 배포 파이프라인 형성 및 데이터 시각화 성공.

Result

설비 이상 징후 보고 및 공유 시간 실시간화. AI 아키텍트 역량을 활용해 기술적 장벽을 빠르게 넘고 비즈니스 가치 창출.

Why: 엔지니어의 도구로서 AI 활용 능력을 극대화하여 결과물의 질을 높이고 리소스를 효율화함.
How: Gemini의 코드를 라인 바이 라인으로 검증하고 Firebase 보안 규칙을 직접 설정함.
